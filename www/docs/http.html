🌐 HTTP Protocol Implementation

Request Parsing

When the server receives raw bytes from a client, the http_parse() function processes them into a structured http_request_t. The parser handles:

  • Request line: METHOD, path, and HTTP version extracted via sscanf
  • Headers: Key-value pairs split on ":" with whitespace trimming
  • newlines: Both \r\n and \n line endings are handled
  • Malformed requests: Returns -1 on parse failure, triggering a 400 response

A typical HTTP request looks like this:

GET /index.html HTTP/1.1
Host: localhost:8080
User-Agent: curl/8.0
Accept: */*

Response Building

The http_build_response() function constructs a complete HTTP response in a buffer:

HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1024
Connection: close
Server: mini-httpd/0.2

<html>...body data...</html>

Static File Serving

The static_serve() function handles the full lifecycle of serving a file:

  1. URL decoding (%20 → space)
  2. Query string stripping
  3. Path safety checks (no .., no absolute paths)
  4. Directory index resolution (index.html or directory listing)
  5. File reading with 16MB size limit
  6. MIME type detection from file extension
  7. Response assembly with Content-Type + Content-Length

Directory Listing

When a URL maps to a directory without an index.html, the server generates an HTML page listing all files with icons, human-readable sizes, and last-modified dates. The directory listing supports recursive navigation through parent directory links.

/api/status Endpoint

This special endpoint returns a JSON object with live server metrics, used by the dashboard page for real-time monitoring:

{
  "version":   "0.2",
  "requests":  42,
  "bytes":     123456,
  "uptime":    3600,
  "uptime_str": "1h 0m 0s",
  "status":    "running"
}

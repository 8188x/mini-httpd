🏗️ Architecture Overview

Server Lifecycle

main() ├─ parse CLI args (-p port, -r root) → server_init() │ ├─ socket() → bind() → listen() │ └─ kqueue() → register listening fd ├─ server_run() │ └─ kevent() loop: │ ├─ new connection → accept() → register in kqueue │ └─ client readable → handle_client() │ ├─ read() → http_parse() │ ├─ static_serve() or /api/status │ ├─ http_build_response() │ └─ write() → close() → log └─ server_close()

Module Structure

main.c

Entry point. Parses command-line arguments (-p port, -r root). Sets up signal handlers for SIGINT/SIGPIPE. Initializes and runs the server.

server.c / server.h

TCP socket initialization and kqueue-based event loop. Manages the listening socket, accepts new connections, and dispatches client events to the HTTP handler. Exposes global server stats.

http.c / http.h

HTTP/1.1 protocol implementation. Parses raw request bytes into structured request objects. Builds full HTTP responses with proper headers. Handles the /api/status endpoint. Generates styled error pages. Writes colored access logs to stderr.

static.c / static.h

Static file serving and directory listing. Maps URL paths to filesystem paths with security checks. Auto-generates HTML directory listings with file icons, sizes, and dates. Enforces 16MB file size limit.

mime.c / mime.h

MIME type detection. Maps 25+ file extensions to Content-Type headers using a lookup table with strcasecmp for case-insensitive matching.

Event Loop Design

The server uses a single-threaded, event-driven architecture based on kqueue (macOS/BSD). Unlike thread-per-connection models, this approach uses a single thread to manage thousands of concurrent connections with O(1) event notification complexity.

The event loop is simpler than select/poll because kevent only returns ready events rather than requiring a full scan of all file descriptors. Edge-triggered mode is available but the server uses level-triggered for simplicity and reliability.

Security

The server implements three layers of path security: absolute path rejection (paths starting with / after stripping the leading / of the URL are blocked), directory traversal prevention (".." in the path is blocked via strstr), and null byte protection (strnlen check ensures paths can't be truncated).
